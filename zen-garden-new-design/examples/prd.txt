# Product Requirements Document: Widgeto Dashboard

## Overview
Widgeto Dashboard is a financial management and activity tracking application that provides users with a comprehensive overview of their financial activities, transactions, earnings, and scheduled tasks. The dashboard presents key metrics, visualizations, and transaction histories in an intuitive, card-based layout.

---

## 1. Navigation Sidebar

### 1.1 Brand Identity
**Component:** Logo and Brand Name
- **Function:** Displays the Widgeto brand identity with a purple circular icon containing a stylized globe design
- **Elements:**
  - Purple circular logo icon (44x44px)
  - "WIDGETO" text in bold purple typography
  - "DASHBOARD" subtitle in smaller purple text
- **Purpose:** Establishes brand recognition and provides visual anchor for the application

### 1.2 Navigation Menu
**Component:** Vertical navigation menu with 7 items
- **Function:** Provides primary navigation throughout the application
- **Menu Items:**
  1. **Dashboard** (Active) - Main overview page (currently selected, indicated by purple vertical bar)
  2. **Activity** - Activity tracking and history
  3. **Library** - Resource or document management
  4. **Security** - Security settings and configurations
  5. **Schedules** - Calendar and scheduled events
  6. **Payouts** - Payment and payout management
  7. **Settings** - Application configuration and preferences
- **Visual Indicators:**
  - Each item has an icon representing its function
  - Active state shown with bold text, darker color, and vertical purple accent bar on the right
  - Inactive items display in lighter gray text
- **Behavior:** Click to navigate to respective sections

## 7. Visual Design Principles

### Color Scheme:
- **Primary:** Purple (#4318FF) - Brand color, accents, charts
- **Background:** Light gray/blue (#F4F7FE)
- **Cards:** White (#FFFFFF)
- **Success/Positive:** Green (#05CD99)
- **Negative/Warning:** Red/Pink (#EE5D50, #E31A1A)
- **Text Primary:** Dark blue-gray (#1B2559, #2B3674)
- **Text Secondary:** Medium gray (#707EAE, #A3AED0)

### Typography:
- **Primary Font:** DM Sans
- **Accent Font:** Poppins (logo only)
- **Hierarchy:** Clear size differentiation between headings, values, labels, and body text

### Layout:
- **Grid-based:** Responsive card layout
- **Spacing:** Consistent 20px gaps between cards
- **Cards:** Rounded corners (20px border-radius), subtle shadows
- **Responsive:** Adapts to different screen sizes with breakpoints

---

## 8. User Interactions

### Expected Behaviors:

1. **Navigation Items:** Click to navigate to different sections
2. **Dropdown (Monthly):** Click to change time period for balance view
3. **Search Box:** Type to search, Enter to execute search
4. **View All Links:** Navigate to detailed views of transfers and tasks
5. **Card Hover States:** Subtle elevation/shadow changes on hover
6. **Menu Icon (Credit Balance):** Open options for credit management
7. **Chart Button (Earnings):** Toggle chart display or settings

---

## 9. Data Requirements

### Real-time Data:
- Current activity values
- Balance and savings percentages
- Recent transfers (with timestamps)
- Credit balance
- Monthly earnings

### Historical Data:
- Trend charts require time-series data for:
  - Activity trends
  - Balance history
  - Earnings patterns
  - Spending patterns

### User Data:
- Username for personalization
- Avatar images for transfers (currently placeholder)
- Transaction categories and icons
- Schedule/calendar events

## 10. Technical Considerations

### Performance:
- Charts should render efficiently with animation
- Lazy loading for historical data
- Responsive image handling for avatars

### Accessibility:
- Semantic HTML structure
- ARIA labels for interactive elements
- Keyboard navigation support
- Color contrast compliance (WCAG 2.1 AA)

### Responsive Design:
- Mobile: Vertical stack layout
- Tablet: 2-column grid
- Desktop: Multi-column grid (current design)
